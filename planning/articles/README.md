## Source of truth

Every article is grounded in [methodology/flow.md](../../methodology/flow.md) (v2.0.0, revised 2026-04-24). If the spec changes in a way that alters phase names, axis semantics, handoff shape, or matrix cells, the corresponding article needs a sweep.

## Recommended publishing order

1. **Article 1 (flagship) first** — establishes vocabulary the others rely on.
2. **Article 2 (designers)** second — primary audience; strongest narrative hook (the seven pains).
3. **Article 3 (tech teams)** third — converts the designer-facing pitch into dev-team value.
4. **Article 4 (leadership)** last — the "what should I do about this" piece for decision makers.

Could also publish 1 + 4 as a pair for an exec audience, then 2 + 3 for practitioner audiences, if the distribution channel splits that way.
